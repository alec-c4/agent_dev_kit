# Extending Agent Dev Kit

## Add a new stack

1. Create stack skill profile:

```yaml
# skills/stacks/my-stack/profile.yaml
id: my-stack
label: My Stack
language: typescript
framework: my-framework
tooling:
  test: [npm test]
  lint: [npx eslint .]
skills:
  required: [stack-detection, testing-universal]
  recommended: [my-stack-patterns]
  if_gem:
    some-gem: [some-pattern-skill]
  if_file:
    config/application.rb:
      content_match: api_only
      skills: [my-api-patterns]
  if_package_dep:
    react: [react-patterns]
dod_overlay:
  - id: my_custom_check
    label: Describe stack-specific DoD item
```

Add `skills/stacks/my-stack/SKILL.md` (see existing stacks for template).

2. Register detection only in `registry/stacks.yaml`:

```yaml
detection_order:
  - my-stack

stacks:
  my-stack:
    detect:
      files: [package.json]
      package_dep: my-framework
    stack_skill: stacks/my-stack
```

3. Add topic files in `registry/topics.yaml` if needed (point to skills, not inline docs).

4. Optional deep pattern skill: `skills/my-stack-patterns/SKILL.md` — reference from profile `skills.recommended`.

5. Compile:

```text
./scripts/kit compile
./scripts/kit detect-stack /path/to/app --write-profile
./scripts/kit validate --phase=1
```

## Add a cross-cutting topic

In `registry/topics.yaml`:

```yaml
topics:
  my_topic:
    skill: my-skill
    stack_files:
      rails: my-skill/stacks/rails.md
      node: my-skill/stacks/node.md
```

Stack-specific topic content lives under `skills/` — not in guidelines.

## Add a pack (install)

1. Create `packs/my-pack/manifest.yaml` with `depends_on: ["core"]` and a `skills` list.
2. Add skills under `skills/`.
3. Run `./scripts/kit compile` and `./scripts/kit validate-skills --pack=my-pack`.
4. Install: `./scripts/kit install --pack=core,my-pack` or `./scripts/kit deploy-skills --pack=my-pack`.

Community packs: copy [packs/community/_template/](packs/community/_template/) → `packs/community/<id>/`.

Official stack packs: `rails`, `node`, `python`, `go`, `elixir`, `devops`, `astro`, `tauri`, `swift`, `kotlin`, `react-native`, `flutter` — see [packs/README.md](../packs/README.md).

## Project-level overrides

In your application repo:

```
my-app/.claude/
├── CLAUDE.md              # Optional stack declaration
└── stack.profile.json     # Generated by detect-stack.sh
```

## Do not

- Put tooling commands or framework conventions in `docs/guidelines/` — use stack skills.
- Bloat `registry/stacks.yaml` beyond detection + `stack_skill`.
- Duplicate tooling in `commands/` — use `stack-loader` and stack profiles.
- Commit profile/registry YAML without running `./scripts/kit compile`.
